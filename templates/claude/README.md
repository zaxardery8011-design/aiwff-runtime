# 人格範本（Persona Templates）

這裡是幾套現成的 AI 人格範本。目的很簡單：讓你**第一天就有一個有個性的助理**，而不是對著空白的 `CLAUDE.md` 發呆。

## 有哪些範本

| 檔案 | 人格 | 適合 |
|---|---|---|
| `persona_dev_partner.md` | 開發夥伴 | 寫程式、修 bug，要直接、程式碼優先 |
| `persona_research_assistant.md` | 研究助理 | 查資料、做比較、要來源與查證 |
| `persona_daily_helper.md` | 生活秘書 | 排行程、列待辦，要親切、清單化 |

## 怎麼用

1. 挑一套最接近你需求的範本。
2. 打開它，把「## 這是誰」以下的內容**複製到你 repo 根目錄的 `CLAUDE.md`**（這個檔會在每次任務時自動載入，成為助理的人格）。
3. 存檔，開始用。助理就會照這個個性回應你。

> 檔案開頭那段引言（`>` 那行）是給你看的說明，貼進 `CLAUDE.md` 時可以不用帶。

## 怎麼混搭客製

範本是起點，不是規定，你可以自由改：

- **換語氣**：不喜歡太直接？把「說話風格」那幾條改成你要的口吻。
- **加規則**：有專屬工作流程，就往「工作習慣」再補幾條。
- **混人格**：例如把「開發夥伴」的程式碼優先，配上「研究助理」的來源標註——直接把兩份的段落挑著拼起來就行。
- **調回報格式**：每套都有一個「做完主動回報」的區塊，改成你團隊慣用的格式。

## 記憶怎麼配合

每套範本都會請助理把你的偏好寫進 `memory/preferences.md`。用久了，助理會越來越懂你的習慣，不用每次重講。你也可以直接手動編輯那個檔，把重要偏好先寫進去。

---

改壞了也沒關係——範本原檔一直在這裡，隨時可以複製一份重來。
