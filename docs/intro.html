What it is

A runtime layer, not a finished assistant.

AIWFF Runtime focuses on durable task execution. It gives developers a small local surface for accepting work, tracking progress, writing artifacts, and proving that a worker finished correctly.

01

Daemon

A local HTTP service on port 3100 that accepts tasks, lists runtime state, serves the cockpit, and owns the task lifecycle loop.

02

File-bus

Tasks, progress logs, and artifacts are persisted as local JSON and JSONL files, so the runtime remains inspectable without a hosted database.

03

Worker

A spawned process receives a task, reports progress, and writes the final artifact. Phase 1 ships a mock worker that completes in about three seconds.

Architecture

Browser, daemon, file-bus, worker.

The current implementation keeps the whole loop local and observable. The cockpit and CLI talk to the daemon; the daemon writes task files; the worker updates progress and artifacts.

Input

Browser or CLI

Submit a task or inspect runtime status through the cockpit and HTTP endpoints.

Control

HTTP daemon

Accepts task records, normalizes state, and exposes health, task, event, and progress APIs.

State

Local file-bus

data/tasks, progress JSONL, and data/artifacts form the durable local source of truth.

Execution

Worker process

Runs the task lifecycle and writes a result that can be checked by npm run verify-demo.

Roadmap

Honest phases, testable growth.

The project starts with a mock-first runtime, then adds real command-line LLM workers, then optional chat interfaces once the execution layer is proven.

Phase 1 complete

Mock-first runtime

HTTP daemon, pending to running to done or failed task queue, local file-bus, cockpit WebUI, mock worker, and verification script.

Phase 2 next

Claude / Ollama workers

Connect real command-line LLM workers so local tasks can move beyond demo lifecycle into useful execution.

Phase 3 later

Telegram / LINE interface

Add optional conversation surfaces after the local runtime is stable, observable, and safe to operate.
